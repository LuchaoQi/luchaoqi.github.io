Manage subrepo without changing history

A lot of people already know how to add a subrepo using submodule and delete a submodule. But that’s how we manage “external” repos. (btw sometimes I just simply add the path of .git file under subrepo into the .gitignore file in the root folder)

In the case we want to move our own repo into another root repo as a subrepo without losing history. I’ve tried a lot of methods and so far the following method I found works well and pretty convenient.

Add the following code into your .bash_profile. In linux, edit .bashrc.

source your .bash_profile/.bashrc

and then

.gitignore

I usually do setting the gitignore file and syncing with github separately since I found git add . sometimes doesn’t work even though I have set up the .gitignore file correctly. To solve this, use git add $path to the specific files supposed to be ignored.

Check out the tutorial by atlassian for additional resources including ignoring syntax, ignoring a previously committed file, etc.
